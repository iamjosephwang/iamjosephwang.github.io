Exception in python

In my practice with respect to ‘I/O manipulation in python’, I try to use ‘try, except’ pairs to make my code throw exception when there’s no existing ‘.csv file’

Exceptions are those errors that the user can make. By default a Python programme will crash when an exception occurs.
In that case, it’s wise for us to indicate where an exception might occur in order to catch it.
We deploy the movement of ‘throwing exceptions’ by using ‘try, except’ pair
The code that might cause an exception that we want to catch is placed inside the ‘try’ block

When an exception occurs inside a try block, execution immediately jump to ‘except’ block. In my practice, we the .csv file cannot be find in the specified directory, the execution will jump to throw ‘FileNotFoundError’
