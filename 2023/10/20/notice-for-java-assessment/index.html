Java Basics

statically typed vs. dynamically typed

Java is statically typed,which means the data type of variables in java are known and checked at compile time, before the program is executed.(vs. dynamically typed language like python)
types of all variables must be declared(unlike python)
Local variables, class fields, method parameter and return values.

Variables cannot be redeclared with a different type

String - not primitive, immutable

to store a value of type t1 in a variable of type t2, the value must be converted to t2 before the assignment occurs.

implicit conversion vs. explicit conversion

  1. implicit(widening) conversion: precision may be lost, but magnitude of numeric value is preserved

  2. explicit(narrow) conversions - casting: may cause significant information loss(narrow conversions must be made explicit)

  3. How does the cast work?
    double to float: loss precision
    out-of-range values become infinite
    some non-zero values may become zero ⭐
    Integer type to ‘smaller’ integer type:
    discard all but n lowest-order bits(value mod max)

double/float to long/int/short/…
round-to-zero: fractional part of the value being removed

String <-> primitive types

  1. converting from Strings: use parseXXX methods
  2. convert to strings:String.valueOf() or use concatenate with an existing String-type variable

division operator:

either of the component is floating-point, it does floating division

scope and accessible

Accessible w.r.t variable in scope: from the point of its declaration to the end of enclosing block

for loop vs. while loop

  1. for loop contains 3 clauses:
    intialization, termination condition check, variable update
    termination check occurs before the loop executes

  2. in the context of while loop, loop variable(the variable you use to control loop) is available before and after loop

  3. advanced in loop:
    what if you want to skip the rest of the loop execution:
    what if you want to break out the entire loop workflow immediately:

switch statements:

  1. if: evaluates any boolean expression
  2. switch: evaluates integer(byte,short,int,….,char,enum) or String(one or more cases)

method in java

  1. A method has 6 components: access modifiers, return type, method name, parameter list, exception list, body
  2. Calling the method, use the name, passing parameters into the parentheses, store the return value (not need when method is void)
  3. It is possible to have two methods with same name as long as they have different parameter lists.(overloading)

Declaring an array:

  1. each array has a type and a dimension
  2. String[] args and String args[] both make sense
  3. subscripts start at zero
  4. An array’s length cannot be changed after it is initialized.
  5. attempt to use an out-of-range index will occur an error

Object and Inheritance

Components - Variables and Methods

Three types of variable

  1. local variables(defined within the method scope)
  2. method parameters(in method headers)
  3. member variables in a class - known as field

constructors

something like a method with the same name as the class
No return type specified
Sets up initial values for the data fields to initialise object state
Use this keyword to refer to the current object being created

visibility modifiers

getters and setters - give controlled acess to private properties.

Inheritance - in java, a class can have only one superclass

What can you do in a subclass(Fields)
Use the inherited fields if they are not marked as private
Declare a field in the subclass with the same name as in the superclass(hiding parent field is not recommended)
Declare new fields.

overriding a method means: implement a method with same signature in superclass but may have different behaviour

Abstract, final, interface

final indicates that something is unchangeable or constant

1.class marked as final

for some security implication
cannot be subclassed

2.final variable

  1. final variable - can never be changed as long as being initialized
    you are highly recommended to name a final variable in a way that combines uppercase letters and underscores(to indicate it’s a constant)
  2. field declared by using final - the value can only be set at declaration time or in a constructor.
  3. final parameter - cannot be changed within a method
    static final variable
    create a constant that is associated with the class itself rather than with instance of class.

3.method marked as final

cannot be overriden or modified by other subclasses

4.class marked as abstract

  1. must implement relevant abstract method in the subclass.
  2. if the subclass does not implement all the abstract method, it must also be marked as abstract.
  3. methods that must be overriden are also need to mark as abstract

5.interface in Java

interface represents class relationships outside the main heritance hierarchy

Declared with the keyword ‘interface’:

  1. all non-static methods are implicitly public abstract
  2. all fields are implicitly public static final
  3. comes after extends

you cannot directly create an instance of interface

Interface vs. abstract class

Interface:

  1. cannot be instantiated(like abstract class)
  2. has no constructor(abstract class can have constructor)
  3. All methods are public(methods can have any access modifier)
  4. Contain only constant fields(“constant and normal fields“)
  5. Classes can implement multiple interface(can have at most one parent class)

Exceptions in Java

Unchecked Exceptions vs. Checked Exceptions

  1. Unchecked Exceptions
    do not need to be explicitly handled
    like ArrayIndexOutOfBoundsException, NullPointerException

  2. Checked Exceptions
    Must be explicitly handled.

Ways of handling exceptions

equals() vs. hashCode()

if you override equals(), you must also override hashCode()
otherwise, it may not run correctly as you expected.
